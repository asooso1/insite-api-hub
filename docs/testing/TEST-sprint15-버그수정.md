# Sprint 15 버그 수정 테스트 체크리스트

## 테스트 대상
- 테마 토글버튼 동작
- 다크모드 컴포넌트 적용
- 중복 검색바 제거
- 통계 카드 hover 배경색
- input 입력 중 키보드 네비게이션

---

## 1. 테마 토글버튼 (다크/라이트/시스템 3가지 옵션)

### 테스트 URL
- `http://localhost:3000` (DashboardV2 헤더)

### 테스트 절차
1. 페이지 상단 헤더의 테마 토글 버튼 확인
2. 버튼에 마우스 hover 시 드롭다운 메뉴가 나타나는지 확인
3. 드롭다운 메뉴에 "라이트", "다크", "시스템" 3가지 옵션이 있는지 확인
4. "다크" 선택 시 다크모드로 전환되는지 확인
5. "라이트" 선택 시 라이트모드로 전환되는지 확인
6. "시스템" 선택 시 OS 설정에 따라 테마가 적용되는지 확인

### 예상 결과
- [ ] 드롭다운 메뉴가 정상 표시됨
- [ ] 3가지 옵션이 모두 표시됨
- [ ] 각 옵션 클릭 시 테마가 즉시 변경됨
- [ ] 선택된 테마는 페이지 새로고침 후에도 유지됨

---

## 2. 다크모드 컴포넌트 적용

### 테스트 URL
- `http://localhost:3000` (메인 대시보드)
- `http://localhost:3000/teams`
- `http://localhost:3000/projects`

### 테스트 절차
1. 다크모드로 전환
2. 헤더 배경색 확인 (어두운 배경이어야 함)
3. 사이드바 배경색 확인
4. 메인 컨텐츠 영역 배경색 확인
5. 카드 컴포넌트들의 배경색 확인
6. 텍스트 색상이 읽기 편한지 확인
7. 버튼 스타일 확인

### 예상 결과
- [ ] 헤더: 어두운 배경 (slate-900)
- [ ] 사이드바: 어두운 배경
- [ ] 메인 영역: 어두운 배경 (slate-950)
- [ ] 카드: 어두운 배경에 밝은 테두리
- [ ] 텍스트: 밝은 색상으로 가독성 확보
- [ ] 버튼: 적절한 대비를 가진 색상

---

## 3. 중복 검색바 제거

### 테스트 URL
- `http://localhost:3000`

### 테스트 절차
1. `Cmd+K` (Mac) 또는 `Ctrl+K` (Windows) 누르기
2. 검색 모달이 **하나만** 열리는지 확인
3. 검색어 입력 후 결과 확인
4. ESC 또는 모달 바깥 클릭으로 닫기
5. 헤더의 검색 버튼 클릭 시 같은 모달이 열리는지 확인

### 예상 결과
- [ ] 검색 모달이 하나만 열림 (중복 없음)
- [ ] 검색 기능 정상 동작
- [ ] 키보드 단축키와 버튼 클릭 모두 같은 모달을 열음

---

## 4. 통계 카드 hover 배경색 제거

### 테스트 URL
- `http://localhost:3000/?tab=endpoints`

### 테스트 절차
1. 상단의 통계 카드들 확인 (전체 엔드포인트, 데이터 모델, 활성환경, 테스트 성공률)
2. 각 카드에 마우스 hover
3. hover 시 불필요한 색깔 변화가 없는지 확인

### 예상 결과
- [ ] hover 시 과도한 배경색 변화 없음
- [ ] 미묘한 3D tilt 효과만 적용됨
- [ ] 깔끔하고 professional한 hover 효과

---

## 5. input 입력 중 키보드 네비게이션 비활성화

### 테스트 URL
- `http://localhost:3000` (검색 모달 또는 어떤 input 필드)

### 테스트 절차
1. 아무 input 필드에 포커스 (검색바, 환경 설정 등)
2. 숫자 1, 2, 3, 4, 5, 6 입력
3. 탭이 변경되지 않고 숫자가 입력되는지 확인
4. `Cmd+K` 단축키는 여전히 동작하는지 확인

### 예상 결과
- [ ] input 필드에서 숫자 키 입력 시 탭 변경 안 됨
- [ ] 숫자가 정상적으로 입력됨
- [ ] `Cmd+K` 같은 modifier 키 조합은 여전히 동작함
