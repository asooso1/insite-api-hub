> spring 2.x 서버 여러개에서 나오는 API 들의 공유를 최적화 하기위해서 공유 플랫폼을 만들건데 몇가지 사항을 
기준으로 계획을 작성해봐 
- git 주소를 넣으면 직접 clone해서 해당 repo의 API 목록들을 request, response까지 상세하게 작성
- git에서 webhook을 수신하면 해당 commit의 diff를 분석해서 api 버전 업데이트
- VO, DTO를 직관적으로 계층을 확인할 수도 있고 클릭해서 펼칠 수도 있음
- repo별, branch별, 프로젝트별(팀 내 진행 프로젝트) 등 필터링이 가능하도록
- 팀 내 프로젝트 관리도 가능하게끔 문서 입력 및 작성이 가능하게끔
- 테스트 케이스 작성 및 테스트 환경을 web에서 실제 Frontend에서 호출하는것처럼 자동생성
- 3개의 서버(dev,stg,prd)를 기준으로 각각 다른 환경 제공(토큰, endpoint 등)
- DB는 현재 개발서버 postgresql 보유중이나 supabase도 무관, 생산성 고려해서 다양한 제안할 것
- 엑셀 형태로 출력해서 공유 및 보관이 쉽도록
- 필수/선택 등의 요소 직관적 표시
- NHN Dooray Api 사용해서 챗봇까지 구현 가능하도록(변경사항, 질문 대답 등)
- FE에서 typescript 사용할 때 자동으로 타입생성할수있도록 
- 코멘트/질문 이력 남길 수 있도록
- 담당자 정보 알 수 있도록
